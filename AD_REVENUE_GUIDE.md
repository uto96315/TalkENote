# 広告収益ガイド

## 実装内容

1. **録音完了時のリワード広告（動画広告）**
   - 録音が完了したタイミングで自動的に表示されます
   - ユーザーが広告を見終わると報酬が付与されます

2. **記録リストのバナー広告**
   - 記録リストで3つに1つ横長のバナー広告を表示します
   - パターン: アイテム0, 1, 2, 広告, アイテム3, 4, 5, 広告...

## 収益の目安

### 一般的な収益指標

**MAU（月間アクティブユーザー）10,000人の場合の収益目安：**

#### リワード広告（動画広告）
- **表示回数**: ユーザー1人あたり月5回録音と仮定 → 50,000回/月
- **表示率**: 70%と仮定（広告が準備できている場合） → 35,000回/月
- **eCPM（1000インプレッションあたりの収益）**: $5-15（日本市場）
- **収益目安**: 35,000回 × $0.005-0.015 = **$175-525/月（約2.6-7.9万円）**

#### バナー広告
- **表示回数**: ユーザー1人あたり月20回記録リストを開くと仮定 → 200,000回/月
- **表示率**: 90%と仮定 → 180,000回/月
- **eCPM**: $1-3（日本市場）
- **収益目安**: 180,000回 × $0.001-0.003 = **$180-540/月（約2.7-8.1万円）**

#### 合計収益目安
- **合計**: $355-1,065/月（約5.3-16万円）
- **年間**: $4,260-12,780（約64-192万円）

### 実際の収益に影響する要因

1. **ユーザーの行動パターン**
   - 録音頻度
   - アプリの使用頻度
   - 広告を見る意欲

2. **広告の種類と品質**
   - リワード広告は通常、バナー広告より高単価
   - 動画広告は完了率が高いほど収益が上がる

3. **地域と市場**
   - 日本市場は比較的高単価
   - アメリカ市場はさらに高単価の可能性

4. **アプリのカテゴリ**
   - 教育・学習アプリは比較的高単価
   - ターゲット層によって広告単価が変わる

5. **広告の最適化**
   - AdMobの自動最適化を有効にする
   - 広告の配置を調整して視認性を上げる

### 注意点

- 上記の数値は**あくまで目安**です
- 実際の収益は様々な要因によって大きく変動します
- 初期段階では収益が低い可能性があります
- AdMobの収益化が承認されるまで時間がかかる場合があります

## 次のステップ

1. **AdMobアカウントの作成**
   - [Google AdMob](https://admob.google.com/)でアカウントを作成
   - アプリを登録して広告ユニットIDを取得

2. **広告ユニットIDの設定**
   - `lib/service/ad_service.dart`の`_productionRewardedAdUnitId`と`_productionBannerAdUnitId`を実際のIDに置き換え
   - `android/app/src/main/AndroidManifest.xml`のApp IDを実際のIDに置き換え
   - `ios/Runner/Info.plist`のApp IDを実際のIDに置き換え

3. **テスト広告の確認**
   - 現在はテスト広告IDが設定されているため、開発中でも広告が表示されます
   - 本番環境では必ず実際の広告ユニットIDに置き換えてください

4. **収益のモニタリング**
   - AdMobダッシュボードで収益を確認
   - ユーザーの行動を分析して広告配置を最適化
